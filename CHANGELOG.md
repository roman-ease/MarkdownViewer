# Changelog

## [1.1.4] - 2026-03-18
### Fixed
- アプリ起動中に別のファイルをダブルクリックした際、同じファイルが無限にタブとして追加され続ける重大な不具合を修正。
  - 原因となっていたIPCポーリングの競合状態（競合読み込み）を解消し、メッセージ処理の排他性を確保。


## [1.1.3] - 2026-03-18
### Added
- シングルインスタンス化（二重起動防止）機能を実装。既にアプリが起動している場合、ダブルクリックされたファイルは既存ウィンドウの新しいタブとして開かれるよう改善。

### Fixed
- ウィンドウを前面に出す（フォーカスを奪う）処理の安定性を向上。


## [1.1.2] - 2026-03-18
### Fixed
- mdファイルのダブルクリック（起動引数）で、複数のファイルを一度に開いた際にすべてタブとして開かれるよう修正。
- ローカルの絶対パス（C:\...）や `file://` 形式で記述された画像が表示されない不具合を修正。
- ファイル外部変更監視が Neutralinojs の仕様変更により機能していなかった不具合を修正（modifiedTimeの利用）。


## [1.1.1] - 2026-03-18
### Fixed
- 画像の相対パス読み込みが非同期的に正しく描画されない不具合を修正。
- ドラッグ＆ドロップでファイルを開いた際に新規タブが作成されなかった問題を修正。
- 「ファイルを開く」ダイアログで複数ファイルを選択した際に、最初の1つしか開かなかった問題を修正。
- 大容量画像のBase64変換時にブラウザがフリーズ・クラッシュする可能性を低減（チャンク処理の導入）。


## 2026-03-13

### Added
- **Markdown Viewer MVP の初期実装**
  - Neutralinojs（システム WebView2 利用の軽量デスクトップアプリフレームワーク）でプロジェクトを構築
  - 左右2ペインUI（エディタ + リアルタイムプレビュー）
  - ビューのみ表示モード（エディタを隠してプレビューのみ表示）
  - ファイル操作: 新規作成 / 開く / 保存 / 名前を付けて保存
  - キーボードショートカット: Ctrl+N, Ctrl+O, Ctrl+S, Ctrl+Shift+S
  - 起動引数によるファイルオープン（.md ファイル関連付け対応）
  - 未保存変更の確認ダイアログ（閉じる前に警告）
  - ファイル読み込み/保存失敗時のエラー表示（アプリは落とさない）
  - ファイル外部変更検知（3秒ポーリング）と再読み込み通知バー
  - ペインのドラッグリサイズ
  - ダークテーマベースのモダンUI
  - Markdown レンダリング: marked.js（GFM対応）
  - ステータスバー（ファイルパス、エンコーディング、行数表示）
- **理由**: Windows向け軽量Markdownビューア/エディタのMVPとして
- **影響範囲**: 新規プロジェクト全体

### Added (Phase 2)
- **ドラッグ&ドロップ対応**: エディタ画面へのファイルドロップで開く機能を追加
- **履歴機能**: 最近使ったファイルをツールバーのドロップダウンから選択可能に（最大10件）
- **シンタックスハイライト**: highlight.js を用いてコードブロックを色付け
- **相対パス画像表示**: Markdownフォルダと同階層の画像をローカル読み込みして表示
- **HTMLエクスポート機能**: 現在のプレビュー状態を独立したHTMLファイルとして保存可能に


### Fixed
- **ドラッグ&ドロップ不具合の制限事項への変更**: Chromium(WebView2)のセキュリティ制限により、OSからドロップされたファイルの絶対パスが取得できないことが判明したため、標準のHTML5 File APIを利用して「ファイルの中身だけ」を読み込み、新規（パス未保持）ファイルとして開く仕様にフォールバックしました。（D&Dしたファイルは上書き保存等ができない制限があります）


### Changed
- **運用・ドキュメント**: .md ファイルをダブルクリックで起動するための Windows 標準のファイル関連付け手順をマニュアル化（walkthrough.md 追記）


### Added (Phase 3)
- **PDFエクスポート機能**: ツールバーの「エクスポート」ボタンをドロップダウンメニュー化し、HTMLまたはPDFでの書き出しを選択可能に変更。OS標準の印刷機能（window.print）と専用CSS（@media print）を利用し、外部ライブラリなしで文字選択や検索が可能な高品質PDF出力を実現した。


### Changed
- **Ver 1.0 リリース準備**: 他のWindows PCでも利用・配布できるよう、実行に必要なファイル群と `README_ja.txt` を同梱した配付用パッケージ `MarkdownViewer_v1.0.zip` を作成し、プロジェクトを完成とした。


### Fixed
- **DevToolsの自動起動を無効化**: 実運用向けに `neutralino.config.json` の `enableInspector` を `false` に設定し、アプリ起動時に開発者ツールが自動で開かないように修正した。

### Fixed
- **リリースビルドの修正**: 最新要件に合わせて不要になった「WebView2Loader.dll」の同梱手続きを削除し、正常に `MarkdownViewer_v1.0.zip` が生成されるよう対応。また README からも該当の記載を削除。

### Added (Ver 1.1)
- **提案**: Ver 1.1（Phase 4）の開発要件（Mermaid対応、画像リンク拡張、タブ機能）を実装計画書に追記し、仕様定義を開始した。
