# Changelog

## [1.3.0] - 2026-03-18
### Added
- IPC（プロセス間通信）の高速化。ストレージ監視間隔を `200ms`（従来の 4 倍）に短縮し、他ファイルからの遷移を爆速化しました。
- 履歴機能の自動移行処理。旧バージョンでのタイポ (`merkdown`) を自動的に正しいキー (`markdown`) へ引き継ぎます。

### Fixed
- セカンダリインスタンス起動時の終了待機時間を最適化し、ウィンドウの消去速度を向上。
- 不要なデバッグ用コード、重複したドキュメント、未使用の `try-catch` などを徹底的に排除。
- ソースコード全体のクリーンアップと軽量化を実施。

## [1.2.0] - 2026-03-18
### Added
- シングルインスタンス化（二重起動防止）機能を正式実装。既にアプリが起動している場合、ダブルクリックされたファイルは既存ウィンドウの新しいタブとして確実に開かれます。
- IPC（プロセス間通信）の仕組みを `Neutralino.storage` ベースの堅牢な設計に刷新。

### Fixed
- ストレージ操作権限（`storage.*`）の欠落により、特定の環境でタブ追加が失敗していた重大な不具合を修正。
- 一時フォルダのアクセス権やパス解釈（ショートネーム等）に依存しない安定したメッセージングを実現。
- セカンダリインスタンス起動時のウィンドウの「瞬き」現象を完全に抑制。
- ゾンビプロセス（異常終了した前プロセスの残存）による誤判定を防止するPIDチェックロジックの強化。

### Changed
- ユーザーに安心感を与えるため、ファイル受信成功時に洗練されたデスクトップ通知を表示するように変更。

## [1.1.0] - 2026-03-18
### Added
- Mermaid.js によるダイアグラム描画に対応。
- タブベースの複数ファイル同時編集UIを実装。
- ローカルファイル、インターネット上のリンク双方の画像描画に対応。
- 外部ファイル変更監視機能の安定化。

## [1.0.0] - 2026-03-13
### Added
- Markdown Viewer MVP の初期リリース。
